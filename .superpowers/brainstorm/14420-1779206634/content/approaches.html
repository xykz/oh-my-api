Implementation Approach Options

All options use one credential JSON with accounts[], routing_mode in config, and account-level average balancing.

A

Adapter-First Core Refactor

Create account pool + load balancer + region adapters first. Chat, models, image upload, admin account UI all move onto the same abstractions.

Best for correctness. More files touched, but cleanest boundary for China vs International API differences.

B

Compatibility Layer

Keep most current services intact, wrap Current() so it returns a selected account. Add international handling only where needed.

Fastest path. Lower initial churn, but region differences may leak through existing China-specific paths.

C

Two-Phase Rollout

Phase 1: multi-account China-only with routing modes and UI. Phase 2: international adapter and login flow.

Lowest risk per step. But the first phase would not satisfy the full international goal immediately.
