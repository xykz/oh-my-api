## 3. Token 估算精准化

### 3.1 当前状态

全系统使用 `chars/4` 粗糙估算：
- `internal/db/canonical_metrics.go:65-77`：`EstimateMessageTokens()` 核心函数
- `internal/api/canonical_runtime.go`：写入 sidecar metadata 时调用
- `internal/api/anthropic_handler.go`：非流式/流式响应均手动 `len(text)/4`
- `internal/api/openai_handler.go`：同理

### 3.2 目标状态

当上游 Lingma API 返回真实 `usage` 时，优先使用；否则保持 `chars/4` fallback。

### 3.3 数据流

```
上游响应
  → handler 解析 usage
    → 写入 sidecar.metadata["prompt_tokens"]
       sidecar.metadata["completion_tokens"]
       sidecar.metadata["total_tokens"]
    → canonical_metrics.go 读取时优先用真实值
      → fallback: EstimateMessageTokens() (chars/4)
```

### 3.4 各协议提取点

**OpenAI 协议（`/v1/chat/completions`）：**
- 非流式：`response.usage.prompt_tokens`, `response.usage.completion_tokens`, `response.usage.total_tokens`
- 流式 SSE：最后一个 data chunk 可能带 `usage`（OpenAI 标准，Lingma 支持情况待定，有则提取无则 skip）

**Anthropic 协议（`/v1/messages`）：**
- 非流式：`response.usage.input_tokens`, `response.usage.output_tokens`
- 流式 SSE：`message_start` 事件可能带 `usage`，`message_delta` 也可能带

### 3.5 实现位置

在各 handler 中修改：
- `internal/api/anthropic_handler.go`：非流式（约 187 行）、流式（约 400 行）
- `internal/api/openai_handler.go`：找到对应 usage 估算位置
- `internal/db/canonical_metrics.go`：`CanonicalRecordTokenCounts()` 优先读取 sidecar metadata 中的真实值，无则 fallback 到 `EstimateMessageTokens()`

**注意：** `canonical_runtime.go` 的 `persistCanonicalExecutionRecord` 调用时机在 handler 解析响应之后，因此 handler 需将提取的 usage 通过参数传递给 persist 函数。

## 4. 前端清理旧 Models 页面

### 4.1 当前状态

- `/models` 路由：显示"兼容映射"旧视图，顶部提示"新规则请优先使用策略引擎"
- `/policies` 路由：功能完整的策略引擎页面
- 后端 `/admin/mappings`：policy 兼容层 API，仅被 Models 页面调用

### 4.2 改动清单

**前端：**
1. 删除 `frontend/src/pages/Models.tsx`
2. 删除 `frontend/src/App.tsx` 中的 `/models` 路由
3. 新增 `/models` → `/policies` 重定向（`Navigate` 组件）

**后端：**
4. 删除 `internal/api/admin_handlers.go` 中的 `/admin/mappings` 相关 handler：
   - `handleAdminMappingsList`
   - `handleAdminMappingsCreate`
   - `handleAdminMappingsUpdate`
   - `handleAdminMappingsDelete`
   - `handleAdminMappingsTest`
   - `policyFromModelMapping`
   - `modelMappingFromPolicyIfCompatible`
   - `modelMappingFromPolicy`
5. 删除 `internal/api/server.go` 中的 `/admin/mappings` 路由注册
6. 删除 `internal/api/server_test.go` 中的 `/admin/mappings` 相关测试

### 4.3 兼容性

- 用户书签 `/models` 自动跳转到 `/policies`
- `/admin/mappings` API 返回 404，确认无其他调用方

## 5. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 上游 usage 字段缺失或格式不一致 | 中 | fallback 到 chars/4，不报错；加 debug log |
| handler 提取逻辑引入解析错误 | 中 | 单元测试覆盖；提取失败时 fallback |
| 删除 `/admin/mappings` 后有未知调用方 | 低 | 全局搜索确认；GitHub 搜索项目内引用 |
| 前端路由删除后缓存问题 | 低 | 重定向处理；用户刷新即可 |
